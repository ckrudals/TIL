2. DI 란
Dependency Injection의 약자로 의존성 주입을 의미한다.
하나의 객체가 다른 객체의 의존성을 제공하는 기술이다.

에를들어 Car class와 Engine class가 있다 해보자.
Car class 가 실행될려면 Engine class가 인스턴스로 있어야한다.
이렇게 Engine class 가 필요한 것처럼 필요한 class를 의존성이라고한다.

2 -1 DI를 사용하는 이유
- 코드 재사용 기능: 종속 항목 객체를 쉽게 교체할 수 있다.
- 리팩토링 편의성 : 종속 항목은 구현 세부정보로 숨겨지지 않고 노출되어 있어 검증 가능한 요소가 되므로 객체 생성 또는 컴파일 시 확인할 수 있따.
- 테스트 편의성 : 클래스가 종속 항목을 관리하지 않으므로 다양한 모든 사례를 테스트할 수 있다.


di 란 디펜더쉽 인젝션 의존성 주입을 의미한다.
의존성 주입은 하나의 객체가 다른 객체의 의존성을 제공하는 기술이다.

예를들어 Car class와 Engine class가 있다고해보자.
Car classㄱ가 실행되려면 Engine class가 실행되어야하며 인스턴스로 있어야한다.
이렇게 Engine class처럼 어떠한 한 class에서 다른 class가 필요한것을 의존성 이라고한다.

Di를 사용하면

하나의 class에서 모든 종속성을 관리해주지 않아도 되기때문에 테스트에 용의하다.
종속 하목 객체를 쉽게 교체할 수 있다.

oop는 객체 지향 프로그래밍으로 컴퓨터 프로그래밍 패러다임중 하나입니다.
프로그래밍에서 필요한 데이터를 추상화시켜 상태와 행위를 가진 객체를 만들고 그 객체들 간의 유기적인 상호작용을
통해 로직을 구성하는 프로그래밍이다.


oop는 객체지향프로ㅡ래밍으로 프로그래밍 패러다임중 하나ㅣㅂ니다.프로그래밍 시 필요한 데이터를 추상화시켜 상태와 행위를 가진 객체르 만들고 그 객체들로 유기적인 상호작용을 통해 로직을 구성합니다.

oop는 객체지향 프로그래밍이다.
oop에는 class와 메서드, 객체가 있습니다.

차에는 부품이 있습니다. 이 부품이 모여 차가 만들어집니다.
이때 부품은 객체 차는 class입니다.
이 차를 움직일수도 있겠죠 이 움직이는 행위를 메소드 라고합니다.

정의한것
객체 : 클래스에서 정의한 것을 토대로 실제 메모리에 할당된것

객제치향 키워드 5가지
1. 클래스 + 객체
2. 추상화
3. 캡슐화
4. 상속
5. 다형성

추상화
추상 클래스나, 추상 클래스가 갖는 추상메서드를 의미하는것이 아닌 클래스 설계하는 자체를 의미한다.
공통의 속성이나 기능을 묶어 이름을 붙잉는것

캡슐화란
기능과 특성의 모음을 클래스라는 캡슐에 분류해서 넣는것
목적 : 1. 코드를 재수정 없이 재활용하는 것.
        2. 접근 제어자를 통한 정보 은닉

상속 이란
부모 클래스의 속성과 기능을 그대로 이어받아 사용할 수 있게 하는것,
기능을 일부분 변경해야 할 경우 상속받은 클래스에서 해당 기능만 다시 수정해 사용할 수 있다.

다형성 이란 
하나의 함수명, 변수명이 상황에따라 다르게 해석될 수 있는 것이다.
오버라이딩 : 부모클래스의 메서드와 같은 이름, 매개변수를 재정의한다.
오버로딩 : 같은 이름의 함수를 여러개 정의하고, 매개변수의 타입과 개수를 다르게하며 매개변수에 따라 다르게 호출될수 있는것
